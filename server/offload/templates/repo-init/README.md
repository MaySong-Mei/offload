# `.offload/`

This directory was created by [Offload](https://github.com/anthropics/) — a system for remotely controlling local coding agents from a phone client.

## What's in here

- **`init/`** — the onboarding template (this directory). Defines how the AI agent generates the long-term context files below.
- **`context/`** — long-term context about this repo: summary, architecture, conventions, glossary. Generated by an AI agent during onboarding (and re-generated on demand). Future agent runs working on this repo read these files instead of having to re-read the whole codebase from scratch.
- **`topics/`** — work items (features, bugs, refactors). Each topic has a requirement, a plan, runs, and feedback. These are written from the Offload phone client and read by the local agent during execution.
- **`state.json`** — repo-level metadata: when the repo was initialized, with what template version, etc.

## Gitignore

By default the entire `.offload/` directory is gitignored. If you want to share `context/` with your team (so everyone's local agent has the same long-term memory), edit `.gitignore` to un-ignore that subdirectory:

```gitignore
.offload/
!.offload/context/
```

## Re-initializing

After a major refactor, the `context/*.md` files may be out of date. From the Offload phone client, tap "Re-initialize" on this project's card to re-run the onboarding agent and overwrite `context/`.

## Manual editing

You can edit any file under `.offload/` by hand. The agent treats them as authoritative. The phone client reflects what's on disk.
